Detailed Description

Represents a quadridimensional vector to be used for algebra and representation of a point/vertex in 3D space as homogeneous coordinates to simplify transforms.

In a way, the fourth component authorizes or forbids translation on the vector when multiplied by a transformation matrix (position-vector vs velocity-vector for example).

See Mathematics group description for conventions.

In this documentation, we will use the notation [x; y; z; w] to describe a mathematical vector in vector-space R^4 with components of respective values x, y, z and w.

This class is used whenever possible instead of any library vector class to ensure libraries can be changed easily if needed. Converters for different library vector classes are provided in Utils.hpp. Usage of another library vector class should only be done for performance reasons (likely in the Mathematics module where algebraic algorithms are written) and should be properly documented.

double Vector4::getProjectionLengthOn ( Vector4 const &  base) const

Length of this vector once projected on another vector.

base is normalized before the projection so that its length doesn't change the result.

Notice that the projection length has then a meaning in absolute coordinates only (absolute coordinates in which these two vectors are expressed). Base doesn't define a relative scaled coordinate system.

Vector4 & Vector4::operator+= ( Vector4 const &  vectorToAdd)

Adds another vector to this vector.

Element-wise vector sum is used for x, y and z components.

w remains unchanged, unless vectorToAdd's w component is 1, then this' w component will be set to 1. This ensures commutativity of the addition and makes the most sense. Indeed, "a position + a translation = a position" seems sensible.

Parameters
